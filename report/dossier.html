What This Is

BAN's Protocol is a pure HTML5-Canvas survival shooter set in a randomly generated maze. The enemy horde is not driven by random spawns or a fixed difficulty curve — a fuzzy-logic "AI director" reads the player's live state roughly twice a second and decides how aggressive the horde should be, how generously to drop supplies, and what kind of enemies to send.

What changed since the first submission. The original prototype was a single file with 3 inputs (Health, Ammo, Noise) and an 11-rule hand-written base. Across four development phases it became a modular 17-file codebase and the controller grew into a complete 6-input → 3-output → 729-rule (3⁶) Mamdani MIMO system — inputs Health, Ammo, Noise, Pressure, Exposure, Skill; outputs Threat, Supply, Composition — with an adaptive-difficulty input, a parallel Takagi–Sugeno path, and an in-game analytics tool.

The Three Scoring Policies (why 729 rules stay consistent)

6 inputs × 3 fuzzy sets each = 3⁶ = 729 antecedent combinations. Every combination is covered, so the director never hits an undefined state. The rules are not hand-written — each rule's consequents are generated from three transparent scoring policies over the term indices (Low/Depleted/…=0, Medium/…=1, High/…=2). The table is therefore exhaustive and internally consistent.

THREAT → Aggression

0.55(H+A) + 0.8·N − 0.6·P + 1.7·E + 0.6·(Skill−1)
≤ 2.3 → Passive   ≤ 4.6 → Tactical   else Overwhelming
Exposure dominates; Pressure relieves; Skill is centred so an average player sees the original balance.

SUPPLY → Director Aid

(2 − H) + (2 − A) + 0.6·P
≤ 1.6 → Scarce   ≤ 3.0 → Balanced   else Generous
Rises as Health & Ammo fall — help the desperate, starve the comfortable.

COMPOSITION → Horde Mix

0.5(H+A) + 0.7·N + 0.8·E + 0.5·Skill
≤ 2.0 → Swarm   ≤ 3.8 → Mixed   else Heavy
Strong, loud, exposed & skilled players earn heavier, elite-laden waves.

System Flow — the Inference Loop

01
SenseRead live state: Health, Ammo, Noise, Pressure, Exposure, Skill.
02
FuzzifyGrade each input against its 3 membership sets → degrees.
03
EvaluateFire all 729 rules, AND = min, × rule weight.
04
ClipClip each output's consequent sets by rule strength.
05
AggregateCombine clipped sets per output with max.
06
DefuzzifyCentroid → crisp Threat / Supply / Composition (0–100).
07
ApplyDrive spawn rate & speed, drop generosity, enemy-type mix.
08
LoopRepeat ≈ every half-second, all game long.
